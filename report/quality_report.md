# Отчёт о качестве данных

## 1. Введение

**Датасет:** Данные о продажах товаров интернет-магазина  
**Источник:** Файл `ПРОДАЖА_data_(1).xlsx`  
**Размер:** 500 строк × 9 столбцов  

**Предметная область:** розничная торговля электроникой и аксессуарами (наушники, ноутбуки, телефоны, зарядки, чехлы). Данные содержат информацию о транзакциях за период с января 2024 по май 2025 года.

**Условный заказчик:** Отдел аналитики интернет-магазина электроники

**Возможные задачи ИАД:**
1. Прогнозирование выручки по дням/месяцам
2. Кластеризация покупателей по возрасту и способу оплаты
3. Анализ сезонности продаж по городам

## 2. Паспорт датасета

**Размер:** 500 строк, 9 столбцов

| Столбец | Тип | Описание | Уникальных | Пропуски |
|---------|-----|----------|------------|----------|
| id | int64 | Номер транзакции | 500 | 0 |
| дата | datetime64 | Дата покупки | ~500 | 0 |
| товар | object | Наименование товара | 5 | 0 |
| город | object | Город покупки | 5 | 0 |
| количество | int64 | Количество единиц | 1-100 | 0 |
| цена | int64 | Цена за единицу (руб.) | 500-100000 | 0 |
| скидка_% | float64 | Скидка в процентах | 0-15 | 11 |
| возраст | float64 | Возраст покупателя (лет) | 18-150 | 11 |
| оплата | object | Способ оплаты | 3 | 0 |

## 3. Основные проблемы качества данных

| № | Тип проблемы | Описание | Потенциальный риск | Предлагаемое решение |
|---|--------------|----------|-------------------|---------------------|
| 1 | Пропуски | `возраст`: 11 пропусков (2.2% от всех строк) | Невозможно корректно анализировать возрастные группы покупателей | Заполнить пропуски медианным значением (35 лет) |
| 2 | Пропуски | `скидка_%`: 11 пропусков (2.2% от всех строк) | Искажение расчёта итоговой цены и выручки | Заполнить пропуски значением 0% (без скидки) |
| 3 | Выбросы | `возраст` = 150 лет (строка 151) | Нереалистичное значение искажает статистику | Заменить на медиану (35 лет) или удалить строку |
| 4 | Выбросы | `количество` = 100 штук (строка 101) | Сомнительная крупная покупка, возможна ошибка ввода | Проверить с заказчиком, при необходимости заменить |
| 5 | Аномалии | `город` = "СПб" вместо "Санкт-Петербург" | Неединообразное наименование городов | Привести все названия к полному формату |

## 4. План улучшения качества

**Приоритет 1 (критично):**
- Исправить выброс в возрасте (150 лет → медиана 35 лет)

**Приоритет 2:**
- Заполнить пропуски в возрасте медианным значением (35 лет)
- Заполнить пропуски в скидке значением 0%

**Приоритет 3 (можно отложить):**
- Проверить выброс в количестве (100 штук) — возможно, это реальная оптовая закупка
- Нормализовать названия городов ("СПб" → "Санкт-Петербург")

## 5. Краткое резюме

**Что узнали о данных:**  
Датасет содержит 500 записей о продажах за ~16 месяцев. Цены приведены к рыночным значениям (ноутбуки: ~73 500 ₽, телефоны: ~43 700 ₽, наушники: ~8 900 ₽, зарядки: ~2 800 ₽, чехлы: ~1 100 ₽). Данные в целом пригодны для анализа, но требуют минимальной предобработки.

**Основные проблемы:**  
- Пропуски в возрасте и скидке (по 2.2%)
- Один выброс в возрасте (150 лет)
- Один выброс в количестве (100 шт.)

**Чему научились:**  
Провели полный аудит качества данных: выявили пропуски, дубликаты, аномалии и выбросы. Построили визуализации для разведочного анализа (гистограммы, boxplot'ы, круговые и столбчатые диаграммы, график связи признаков). Сформулировали рекомендации по очистке данных и план улучшения с приоритетами.

**Особенности, требующие внимания при дальнейшем анализе:**  
- Возрастная структура покупателей (основная аудитория — 25-50 лет)
- Региональные различия (Москва и СПб — лидеры по продажам)
- Способы оплаты распределены равномерно (~33% каждый)
