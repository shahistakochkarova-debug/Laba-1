# Отчёт о качестве данных

## 1. Введение

**Датасет:** Данные о продажах товаров  
**Источник:** Файл `ПРОДАЖА_data (1).xlsx`  
**Размер:** 500 строк × 9 столбцов  

**Предметная область:** Розничная торговля — записи о продажах электроники и аксессуаров (наушники, ноутбуки, телефоны, зарядки, чехлы)  

**Условный заказчик:** Отдел аналитики интернет-магазина  

**Возможные задачи ИАД:**
1. Прогнозирование выручки по дням/месяцам
2. Кластеризация покупателей по возрасту и способу оплаты
3. Анализ сезонности продаж по городам

## 2. Паспорт датасета

| Столбец | Тип | Описание | Примечания |
|---------|-----|----------|------------|
| id      | int64 | Номер транзакции | Уникальный, без пропусков |
| дата    | datetime| Дата покупки | Диапазон: 01.01.2024 — 14.05.2025 |
| товар   | object | Наименование товара | 5 категорий: Наушники, Ноутбук, Телефон, Зарядка, Чехол |
| город   | object | Город покупки | 5 городов: Москва, СПб, Казань, Екатеринбург, Новосибирск |
| количество | int64 | Количество единиц | От 1 до 100 (есть выброс — 100 шт.) |
| цена    | int64 | Цена за единицу в рублях | Есть отрицательное значение (-500) |
| скидка_%| float64 | Скидка в процентах | Много пропусков (≈15% строк) |
| возраст | float64 | Возраст покупателя | Много пропусков (≈40%), есть выброс 150 лет |
| оплата  | object | Способ оплаты | 3 варианта: онлайн, карта, наличные |

## 3. Основные проблемы качества данных

| № | Тип проблемы | Описание | Потенциальный риск | Предлагаемое решение |
|---|--------------|----------|-------------------|---------------------|
| 1 | Пропуски | `возраст`: ~40% пропусков (строки 11-17, 19-21 и др.) | Невозможно анализировать возрастные группы | Заполнить медианой (35 лет) или удалить строки |
| 2 | Пропуски | `скидка_%`: ~15% пропусков (строки 31-41) | Искажение расчёта итоговой цены | Заполнить 0% (без скидки) |
| 3 | Выбросы | `возраст` = 150 (строка 151) | Нереалистичное значение, портит статистику | Заменить на медиану или NaN |
| 4 | Отрицательные значения | `цена` = -500 (строка 201) | Некорректный расчёт выручки | Удалить строку или заменить на медиану |
| 5 | Выбросы | `количество` = 100 (строка 101) | Сомнительная крупная покупка | Проверить, возможно это ошибка ввода |
| 6 | Аномалии | `город` = "СПб" вместо "Санкт-Петербург" | Неединообразное наименование | Привести к полному названию |
| 7 | Дубликаты | Нет полных дубликатов строк | — | — |

## 4. План улучшения качества

**Приоритет 1 (критично):** Удалить строку с отрицательной ценой (строка 201), исправить возраст 150 лет  

**Приоритет 2:** Заполнить пропуски в возрасте (медианой) и скидке (0%)  

**Приоритет 3:** Проверить выброс "количество = 100" (возможно, опечатка), нормализовать названия городов  

## 5. Краткое резюме

**Что узнали о данных:**  
Датасет содержит 500 записей о продажах за ~16 месяцев. Данные в целом пригодны для анализа, но требуют предобработки. Основные проблемы: **пропуски** (особенно в возрасте) и **выбросы** (отрицательная цена, нереалистичный возраст). После очистки (удаление 1-2 строк, заполнение пропусков) данные будут готовы для задач прогнозирования и кластеризации.

**Особенности, требующие внимания:**  
- Цены и скидки различаются сильно — нужна нормализация для некоторых моделей  
- Есть покупатели всех возрастов от 18 до 50+ лет  
- Популярные города: Москва и СПб
