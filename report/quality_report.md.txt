# Отчёт о качестве данных

## 1. Введение

**Датасет:** ПРОДАЖА_data (данные о продажах интернет-магазина)

**Источник:** Синтетический датасет, сгенерированный для лабораторной работы №1 по курсу "Интеллектуальный анализ данных"

**Размер:** 500 строк × 9 столбцов

**Предметная область:** Розничная торговля электроникой и аксессуарами. Датасет содержит информацию о транзакциях интернет-магазина: какие товары покупали, в каком количестве, по какой цене, со скидкой или без, а также данные о покупателях (возраст, город) и способе оплаты.

**Условный заказчик:** Отдел маркетинга и аналитики интернет-магазина

**Возможные задачи ИАД:**

1. **Сегментация клиентов** – выделение групп покупателей на основе возраста, города, суммы покупок и частоты заказов для персонализированных маркетинговых кампаний

2. **Поиск ассоциативных правил** – выявление устойчивых пар товаров, которые клиенты покупают вместе, для оптимизации размещения товаров и создания рекомендаций

3. **Прогнозирование спроса** – предсказание объемов продаж по месяцам и категориям товаров для планирования закупок и управления складскими запасами

## 2. Паспорт датасета

| Столбец | Тип | Описание | Примечания |
|---------|-----|----------|-------------|
| id | int64 | Уникальный номер транзакции | Есть дубликаты (2 шт.) |
| дата | datetime64 | Дата и время покупки | Диапазон: 01.01.2024 – 14.05.2025 |
| товар | object | Наименование товара | 5 категорий: Наушники, Ноутбук, Телефон, Чехол, Зарядка |
| город | object | Город покупателя | 5 городов: Москва, СПб, Екатеринбург, Казань, Новосибирск |
| количество | int64 | Количество единиц товара | В основном 1–4 шт., есть аномалия: 100 шт. |
| цена | int64 | Цена за единицу (руб.) | Диапазон: от -500 до 50000 руб., есть отрицательное значение |
| скидка_% | float64 | Размер скидки в процентах | Пропуски (10 шт., 2%), диапазон: 0–15% |
| возраст | float64 | Возраст покупателя (лет) | Пропуски (40 шт., 8%), аномалия: 150 лет |
| оплата | object | Способ оплаты | 3 категории: карта, наличные, онлайн |

## 3. Основные проблемы качества данных

| № | Тип проблемы | Описание | Потенциальный риск | Предлагаемое решение |
|---|--------------|----------|-------------------|---------------------|
| 1 | Пропуски | возраст: 40 пропусков (8%) | Искажение статистики по возрастным группам | Заполнить средним значением по категории товара |
| 2 | Пропуски | скидка_%: 10 пропусков (2%) | Неверный расчет итоговой стоимости | Заполнить нулем |
| 3 | Аномальные значения | цена: 1 запись с отрицательной ценой (-500 руб.) | Некорректный расчет выручки | Удалить строку |
| 4 | Аномальные значения | возраст: 1 запись с возрастом 150 лет | Выброс, искажение статистики | Заменить на медиану (35 лет) или удалить |
| 5 | Дубликаты | id: 2 полных дубликата строк | Двойной учет продаж | Удалить дубликаты |
| 6 | Выбросы | цена: 45 выбросов (9%) по методу IQR | Искажение среднего чека | Проверить; если реальные данные – оставить |
| 7 | Аномальные значения | количество: 1 запись с 100 шт. | Возможная ошибка ввода | Проверить обоснованность |
| 8 | "Грязные" данные | город: разные форматы ("СПб" vs "Санкт-Петербург") | Неверная группировка | Привести к единому формату |

## 4. План улучшения качества

**Приоритет 1 (критично):** Удалить строку с отрицательной ценой (id=201), удалить дубликаты по id (id=1 и id=2), исправить или удалить аномальный возраст 150 лет (id=151)

**Приоритет 2:** Заполнить пропуски в возрасте средним значением по категории товара, заполнить пропуски в скидке нулем

**Приоритет 3:** Проверить обоснованность выбросов по цене, проверить покупку 100 единиц товара (id=101), нормализовать названия городов, добавить новые признаки из даты

## 5. Краткое резюме

**Что узнали о данных:** Датасет содержит 500 записей о продажах товаров за период 2024–2025 годы. Данные включают числовые (цена, количество, возраст, скидка), категориальные (товар, город, способ оплаты) и временной (дата) признаки. Качество данных среднее: выявлены пропуски (возраст – 8%, скидка – 2%), дубликаты (2 шт.) и аномалии (отрицательная цена, возраст 150 лет).

**Какие аспекты требуют особого внимания:** Пропуски в возрасте (8%) критичны для сегментации клиентов. Выбросы по цене (9%) требуют проверки перед удалением. Аномалии должны быть исправлены или удалены до любого анализа. Временной характер данных позволяет решать задачи прогнозирования.

**Итоговая оценка:** После выполнения плана улучшения (особенно приоритетов 1 и 2) данные становятся пригодными для решения задач сегментации, ассоциативного анализа и прогнозирования спроса.
